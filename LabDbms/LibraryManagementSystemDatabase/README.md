# Library Management System - Database Assignment

[cite_start]This repository contains the SQL implementation for the **Library Management System** assignment [cite: 3][cite_start], developed as part of the **Database Management System (BCA37107)** course for the 3rd Semester[cite: 1, 23, 43, 74].

## Project Overview
[cite_start]The objective of this assignment is to design a relational database schema for a library management system and execute various SQL queries to demonstrate proficiency in Data Definition Language (DDL), Data Manipulation Language (DML), and advanced query functions[cite: 32, 91].

## Files Included
* [cite_start]**`LibraryManagementSystem.sql`**: Contains the `CREATE` statements for tables (`Member`, `BOOKS`, `ISSUE`) and the initial data insertion commands[cite: 7, 15, 26, 32].
* [cite_start]**`LibraryQueriesPartOne.sql`**: Contains solutions for the Part-I assignment tasks (Queries 1–33), covering basic retrieval, numeric functions, string functions, and filtering[cite: 47, 55, 61, 68].
* [cite_start]**`LibraryQueriesPartTwo.sql`**: Contains solutions for the Part-II assignment tasks, including aggregate functions, table copying, and schema modifications[cite: 91, 104, 107, 108].

## Database Schema
[cite_start]The database consists of three primary entities[cite: 4, 14, 25]:
1.  [cite_start]**Member**: Stores details about library members, including their membership type and penalty status[cite: 7, 11, 13].
2.  [cite_start]**BOOKS**: Stores bibliographic information, costs, and categories[cite: 15].
3.  [cite_start]**ISSUE**: Tracks the issuance and return of books, linking members and books via foreign keys[cite: 26, 30, 31].

## Prerequisites
* An Oracle SQL environment (or compatible RDBMS).
* Permission to create tables and execute scripts.
